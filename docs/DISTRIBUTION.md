# Distributing this plugin

This is an open-source **Adobe Lightroom Classic** plugin — a Lua `.lrplugin`
bundle. This document is a practical, honest guide to getting it into users'
hands, including the Adobe catalog path and why it may not be the right one for
*this* plugin.

## Recommendation

For this plugin, distribute through **GitHub Releases** as the primary channel
and request a listing in the **[Lightroom Queen plugin directory](https://www.lightroomqueen.com/links/plugins/)**
for discovery. Treat **[Adobe Exchange](https://exchange.adobe.com/apps/browse/cc?product=LTRM)**
as optional and secondary: the submission pathway for Lightroom Classic plugins
still exists but is under-documented, and this plugin's Google Lens behavior
(shelling out to Node.js and driving a visible Chrome window against an
unofficial Google surface) is a genuine review risk — a reviewer who hits a
Google challenge or an unconfigured Node reads the plugin as "does not perform
as described," which is a likely rejection. Off-marketplace distribution is also
simply how most serious Lightroom Classic plugins ship, so leading with it costs
nothing and reaches the same photographers.

## Primary: GitHub Releases

This repository already produces the artifact you'd distribute. The CI workflow
([`.github/workflows/ci.yml`](../.github/workflows/ci.yml)) runs on every push,
but a tag that matches `v*` additionally builds and publishes a **GitHub
Release**: it version-stamps the bundle, zips it, writes `checksums.txt`, and
attaches both. The bundle is **self-contained** — it ships its own per-OS Node
runtime *and* the Lens helper's dependencies (`puppeteer-core`), so users unzip
and run with only Google Chrome: no Node to install, nothing to `npm install`.

To cut a release:

1. Bump the `VERSION` file and add a matching `## [X.Y.Z]` heading to
   `CHANGELOG.md`. (A CI guard fails the tag build if the tag, `VERSION`, and
   changelog heading disagree.)
2. `git tag vX.Y.Z && git push --tags`.
3. CI builds `output/dist/SpeciesTagger.lrplugin-*.zip` + `output/dist/checksums.txt` and
   publishes them to the release, with notes generated by git-cliff.

The install story is the universal one every Lightroom Classic user already
knows, and it matches the [README](../README.md):

- [ ] Download the latest `SpeciesTagger.lrplugin-*.zip` from
      [Releases](https://github.com/yuvaloren/lightroom-species-tagger/releases) and
      unzip it.
- [ ] Lightroom Classic → **File ▸ Plug-in Manager ▸ Add** → select the
      unzipped `SpeciesTagger.lrplugin` folder.

### Get listed for discovery

A GitHub URL is not discoverable on its own. Request a listing at:

- The **[Lightroom Queen plugin directory](https://www.lightroomqueen.com/links/plugins/)** —
  the community's de-facto index of Lightroom Classic plugins.
- Adobe's own **[plug-ins pointer page](https://helpx.adobe.com/lightroom-classic/plug-ins.html)**,
  which links out to third-party plugins rather than hosting them.

## Optional: Adobe Exchange

### Current state

The official Adobe catalog is **Adobe Exchange** (`exchange.adobe.com`), and it
is **not** sunset — the discontinued service was the unrelated "Creative Cloud
Market" asset library. Exchange has a Lightroom Classic section:
<https://exchange.adobe.com/apps/browse/cc?product=LTRM>.

The developer-facing submission side was consolidated into **Adobe Developer
Distribution** (<https://developer.adobe.com/distribute/home>) — UXP plugins
first in 2023, then ZXP/CEP and "non-plugin items" around 2024. Lightroom
**Classic** plugins are the Lua SDK kind (<https://developer.adobe.com/lightroom-classic/>);
Exchange accepts them as a ZIP or a ZXP. The only canonical how-to for
Lightroom Classic specifically is a 2022 Adobe blog post:
<https://blog.developer.adobe.com/en/publish/2022/07/lightroom-classic-plugin-support-for-the-adobe-exchange-for-creative-cloud>.

Two things this pathway is **not**: Lightroom (cloud) has no in-app plugin
marketplace at all (its developer site is a REST API), and the UXP marketplaces
are for Photoshop, InDesign, and Premiere — not either Lightroom.

> **State this plainly:** the current Developer Distribution documentation never
> names "Lightroom Classic." The pathway rests on the 2022 blog post plus the
> later ZXP / non-plugin-items migration. Before investing effort in ZXP
> packaging, **email [ccintrev@adobe.com](mailto:ccintrev@adobe.com)** to confirm
> that Lightroom Classic ZIP/ZXP submissions are still accepted.

### Submission checklist

Once you've confirmed the pathway is open:

- [ ] Sign in with a free **Adobe ID**. There is no developer-program fee.
- [ ] Complete a **publisher profile**. EU trader details have been mandatory for
      EU visibility since **2025-02-17**.
- [ ] Prepare **listing metadata**: a unique plugin name, an English-language
      description, a support email, a live help/support URL, and a privacy
      policy URL (this repo has [docs/PRIVACY.md](PRIVACY.md)).
- [ ] Prepare **assets**: 3 original icons and screenshots.
- [ ] Submit and wait — the review target is roughly **10 business days**.
- [ ] For a **paid** listing, payments run through **FastSpring** at a **90%**
      revenue share to the developer.

Reference docs:

- Overview — <https://developer.adobe.com/developer-distribution/creative-cloud/docs/guides/submission/overview>
- What review checks — <https://developer.adobe.com/developer-distribution/creative-cloud/docs/guides/submission/what-review>
- Timelines and outcomes — <https://developer.adobe.com/developer-distribution/creative-cloud/docs/guides/submission/timelines-outcomes>
- FAQ — <https://developer.adobe.com/developer-distribution/creative-cloud/docs/guides/submission/faq>

## Will the Lens/Chrome behavior pass Adobe review?

The plugin's **assistive design** helps here: it does **not** scrape or extract
Google's results. It opens Google Lens in a visible Chrome window, the **user** reads
the page and highlights the species name, and the plugin uses only that selection —
so it stays within Google's terms (no automated extraction, no bulk access). That
removes the strongest ToS objection a reviewer could raise.

Two practical risks remain, both about *reliability under review* rather than policy:

- If Google throws an IP challenge at Adobe's reviewer (challenges are IP-driven), or
  the reviewer's machine doesn't have Node set up, the window may not load — and a run
  that doesn't complete reads as **"does not perform as described,"** the most common
  rejection reason.
- It's still a **third-party companion app** that shells out to Node.js and drives
  Chrome, which reviewers exercise fully; anything environment-dependent is a risk.

**Assessment:** far more review-friendly than a scraper, but still dependent on the
reviewer's environment and network. It can also **degrade** for an end user whose
network gets an IP challenge (they'd see the check and solve it, or retry later). This
is why the recommendation still leads with GitHub Releases, where the caveats live
right next to the download.

For contrast: John R. Ellis's *Any Vision* uses the **official** Google Vision
API with the user's own key — a far more review-friendly design — and **still**
distributes off-marketplace (see below). That's a useful signal about where
Lightroom Classic plugins actually live.

## How Lightroom Classic plugins are actually distributed

Independent of Adobe Exchange, the established channel is a developer's **own
website or GitHub releases** plus **community directories**. Representative
examples:

- **[Jeffrey Friedl](https://regex.info/blog/lightroom-goodies)** — a long
  catalog of plugins distributed from his own blog.
- **[John R. Ellis](https://www.johnrellis.com/lightroom/)** — including *Any
  Vision*, which uses the **official** Google Vision API with the user's own key
  and still ships off-marketplace.
- **[Photographer's Toolbox](https://www.photographers-toolbox.com/)** — a
  multi-developer storefront with built-in self-update.
- **[Excire](https://excire.com/)** — sold through its own shop.

Discovery for all of these runs through the
**[Lightroom Queen plugin directory](https://www.lightroomqueen.com/links/plugins/)**
and Adobe's own
**[plug-ins pointer page](https://helpx.adobe.com/lightroom-classic/plug-ins.html)**.
The install is universal regardless of source: unzip the `.lrplugin`, then
**File ▸ Plug-in Manager ▸ Add**.
